Omnia Foundation - October 2017

New Features

Omnia Admin Home Extensibility

Possibility to configure which services that should be available in the start page of Omnia Admin using an extension.

Bug Fixes

  • Order static blocks in page layouts did not work as expected.
  • Not possible to add certain site templates in “Disable dynamic properties for these site templates” setting in Omnia Admin.
  • In Safari, the date picker did not close when another date picker was open.
  • Omnia Admin failed to open via the Omnia Admin icon when the security session of a user timed out.
  • The Default button tooltip in Layouts in Omnia Admin was cut off.
  • When opening Omnia Admin using the Omnia Admin icon and then clicking on the breadcrumb in Omnia Admin, the site was opened within the Omnia Admin frame.
  • Site Directory only returned the 50 first sites.

October 2017 Patch 1 (1.0.7802)

  • Omnia Admin icon shortcut doesn't work in environments running in Site Collection Only mode.

October 2017 Patch 2 (1.0.8151)

  • Documentation does not work on SharePoint 2013 in Internet Explorer 11.
  • Owner of Office 365 Group created from Omnia Site Template could not find group in SharePoint search. Owner will now be added to members group also.
  • Some Omnia Sub Site Templates not displayed to end-users except Admins.

October 2017 Patch 3 (1.0.8347)

  • The My Sites navigation didn't show Office 365 groups.
  • Private Office 365 Groups did not show up in the Site Directory.
